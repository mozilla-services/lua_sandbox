3 ## Overview
4 
5 Sandboxes provide a dynamic and isolated execution environment
6 for data parsing, transformation, and analysis. They allow access to data
7 without jeopardizing the integrity or performance of the processing
8 infrastructure. This broadens the audience that the data can be
9 exposed to and facilitates new uses of the data (i.e. debugging, monitoring,
10 dynamic provisioning, SLA analysis, intrusion detection, ad-hoc reporting,
11 etc.)
12 
13 The Lua sandbox is a library allowing customized control over the Lua execution
14 environment including functionality like global data preservation/restoration on
15 shutdown/startup, output collection in textual or binary formats and an array of
16 parsers for various data types (Nginx, Apache, Syslog, MySQL and many RFC grammars)
17 
18 These libraries and utilities have been mostly extracted from [Hindsight](https://github.com/trink/hindsight).
19 The goal was to decouple the Heka/Hindsight functionality from any particular
20 infrastructure and make it embeddable into any tool or language.
21 
22 ### Features
23 
24 - small - memory requirements are as little as 8 KiB for a basic sandbox
25 - fast - microsecond execution times
26 - stateful - ability to resume where it left off after a restart/reboot
27 - isolated - failures are contained and malfunctioning sandboxes are terminated.
28  Containment is defined in terms of restriction to the operating system,
29  file system, libraries, memory use, Lua instruction use, and output size.
